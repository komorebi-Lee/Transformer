# 一阶编码质量标准（扎根理论）

## 1. 定义

一阶编码（开放式编码）是将原始访谈文本提炼为**简短、书面化、语义完整的概念短语**的过程。它不是简单的删词或截断，而是提取句子的核心信息并进行概念化表达。

## 2. 质量标准（五条）

### 2.1 语义完整
- 编码必须是**完整的语义单元**，不能是残缺的短语片段
- 不能以"的、了、着、过、到、在、中、和、与、或"等虚词结尾
- 不能以"因为、所以、但是、如果、虽然、那么"等连词开头
- 不能包含疑问词（什么、怎么、哪些、如何等）——一阶编码是概念陈述，不是问题

**正例：** "客户需求导向不足" → 完整概念
**反例：** "客户需求的" → 语义残缺

### 2.2 重点提取
- 编码必须**抓住句子的核心信息**，而非边缘细节
- 优先提取：问题/困境 > 转折/对比 > 因果链 > 程度/强度
- 句子中具有理论价值的概念（机制、流程、资源、策略等）应优先保留
- 过渡语、寒暄、纯态度表达不应编码

**正例：** "审批流程延迟影响项目推进" → 提取了核心因果关系
**反例：** "就是说可能大概有一些影响" → 空洞无信息

### 2.3 书面化表达
- 编码必须使用**正式的书面语**，不能保留口语特征
- 口语词必须替换为书面词：搞→开展，弄→处理，很难→困难，老是→频繁
- 不能包含语气词（吧、呢、啊、嘛、呀、哦）
- 不能以人称代词开头（我、我们、你、你们、他、他们）
- 不能包含口语填充词（就是说、我觉得、怎么说呢、那个、这个）

**正例：** "资源配置不足导致项目延迟"
**反例：** "我们那个资源不太够然后项目就慢了"

### 2.4 长度适中
- 推荐长度：**4-24 字**
- 最短：4 字（必须是概念完整的专业术语组合）
- 最长：30 字（超过此长度必须缩写或提炼）
- 理想长度：8-18 字

### 2.5 领域覆盖
- 编码应能覆盖以下常见领域的核心概念：
  - **农业**：种植、养殖、加工、销售、土地、气候、品种、产量
  - **经济**：市场、价格、成本、利润、供应链、竞争、消费
  - **手工业**：工艺、技术、生产、质量、标准、培训、传承
  - **组织管理**：管理、制度、流程、审批、协调、团队、领导
  - **技术创新**：研发、改进、方法、设备、自动化、数字化
  - **社会关系**：合作、信任、沟通、网络、社区、互惠

## 3. 编码生成规则

### 3.1 合格编码必须满足
1. 长度 4-30 字
2. 语义完整（不含残缺结构）
3. 书面语（无口语残留）
4. 包含实质性信息（专业术语、因果关系、问题描述等）
5. 与原文有语义关联（模型验证相似度 0.30-0.96 之间）

### 3.2 不合格编码直接淘汰
1. 纯应和（对、嗯、是的、好的）
2. 纯疑问（含什么/怎么/如何等疑问词）
3. 说话人标签（说话人1、受访者等）
4. 长度不足 4 字
5. 语义残缺（以虚词结尾或以连词开头）
6. 口语残留（含语气词、人称代词开头、口语填充词）

## 4. 质量等级

| 等级 | 标准 | 示例 |
|------|------|------|
| A（优秀）| 概念完整、书面化、精准提取核心信息、4-18字 | "审批延迟影响项目推进" |
| B（良好）| 概念完整、基本书面化、提取了主要信息 | "项目推进受审批影响" |
| C（合格）| 概念基本完整、无明显口语、有一定信息量 | "审批影响项目" |
| D（不合格）| 语义残缺/口语化/无实质信息/无关内容 | "审批的影响"（残缺） |

## 5. 准确率计算

一阶编码准确率 = (A级 + B级 + C级) / 总编码数 × 100%

目标：准确率 ≥ 60%（即 D 级不超过 40%）
