### 三、训练参数调整时机与建议

| 训练情况 | 参数调整 | 推荐值 |
|---------|---------|--------|
| **首次训练** | 使用默认参数 | 学习率: 2e-5, 轮数: 3, 批次: 16 |
| **模型过拟合** | 减少轮数、增加dropout | 轮数: 2, dropout: 0.3 |
| **模型欠拟合** | 增加轮数、增大学习率 | 轮数: 5, 学习率: 5e-5 |
| **GPU显存不足** | 减小批次大小 | 批次: 8 或 4 |
| **增量训练** | 降低学习率 | 学习率: 1e-5 (默认的50%) |

## 概述

本系统支持三种训练模式，用于扎根理论编码的自动学习和预测：

| 训练模式 | 说明 | 适用场景 |
|---------|------|---------|
| **BERT微调** | 直接微调BERT模型参数 | 样本充足、追求高准确率 |
| **分类器训练** | BERT特征提取+随机森林分类器 | 样本较少、GPU资源有限 |
| **增量训练** | 在已有模型基础上继续训练 | 持续添加新数据、迭代优化 |

## 快速开始

### 1. 准备训练数据

首先需要通过手动编码创建标准答案：

```python
# 在主界面点击"手动编码"按钮
# 完成编码后，点击"导出为标准答案"
```

标准答案格式示例：
```json
{
  "metadata": {
    "version": "v1_20240101_120000",
    "created_time": "2024-01-01 12:00:00"
  },
  "structured_codes": {
    "C01 组织管理": {
      "B01 团队架构": [
        {"content": "团队职责分工明确", "sentence_details": [...]},
        {"content": "管理层级清晰", "sentence_details": [...]}
      ]
    }
  }

```

### 2. 启动训练

在主界面点击 **"训练模型"** 按钮，选择训练模式：

- **BERT微调**：推荐用于正式训练，效果最佳
- **分类器训练**：快速训练，适合测试
- **增量训练**：在已有模型基础上优化

### 3. 使用训练模型预测

训练完成后，在编码模式中选择 **"训练模型编码"**，系统将自动使用训练好的模型进行预测。

## 训练模式详解

### BERT微调训练

**工作原理**：直接微调BERT模型的全部参数，使其适应扎根理论编码任务。

**配置参数**（可在 `config.py` 中修改）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `FINETUNE_LEARNING_RATE` | 2e-5 | 学习率 |
| `FINETUNE_EPOCHS` | 3 | 训练轮数 |
| `FINETUNE_BATCH_SIZE` | 16 | 批次大小 |
| `FINETUNE_WARMUP_RATIO` | 0.1 | 预热比例 |
| `FINETUNE_EARLY_STOP_PATIENCE` | 3 | 早停耐心值 |

**硬件要求**：
- GPU：建议至少4GB显存
- 内存：建议至少8GB

### 分类器训练

**工作原理**：使用BERT提取文本特征，然后训练随机森林分类器。

**优势**：
- 训练速度快
- 不需要GPU
- 适合小样本场景

### 增量训练

**工作原理**：加载已有模型，在新数据上继续训练。

**特点**：
- 自动降低学习率（默认为原来的50%）
- 合并新旧训练数据
- 记录训练历史

**使用场景**：
```python
# 场景1：添加新标注数据后继续训练
# 场景2：修正模型预测错误后重新训练
# 场景3：持续迭代优化模型
```

## 降级机制

当BERT微调训练条件不满足时，系统会自动降级到分类器模式：

| 降级原因 | 说明 |
|---------|------|
| GPU内存不足 | 显存小于2GB |
| transformers库不可用 | 库未安装或版本不兼容 |
| 训练过程异常 | 训练中出现错误 |

降级时会弹窗提示用户，说明降级原因。

## 最佳实践

### 数据准备

1. **样本数量**：BERT微调建议至少100个样本，分类器建议至少50个样本
2. **类别平衡**：确保各类别样本数量相对均衡
3. **数据质量**：确保标注准确，避免错误标签

### 参数调优

1. **学习率**：从默认值开始，效果不佳时可尝试1e-5或5e-5
2. **批次大小**：GPU显存充足时可增大到32
3. **训练轮数**：观察loss曲线，避免过拟合

### 增量训练策略

1. **时机选择**：新增数据达到10条以上时进行增量训练
2. **学习率**：增量训练自动使用较低学习率
3. **版本管理**：每次增量训练前创建版本备份

## 常见问题

### Q: 训练时提示"训练样本不足"

**A**: 确保标准答案中至少有5个样本。建议BERT微调使用100+样本。

### Q: GPU内存不足怎么办？

**A**: 系统会自动降级到分类器模式。也可手动减小批次大小。

### Q: 如何查看训练效果？

**A**: 训练完成后会显示准确率。也可使用 `model_manager.predict_with_finetuned_bert()` 测试预测效果。

### Q: 如何恢复之前的模型？

**A**: 使用版本管理功能回滚到指定版本。
