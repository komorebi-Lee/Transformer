# 超参数优化算法使用指南

## 概述

本指南介绍了超参数优化系统的新功能，包括多种优化算法、剪枝策略和并行化搜索等特性，帮助用户选择最合适的优化策略，提高模型训练的效率和质量。

## 支持的优化算法

系统现在支持以下超参数优化算法：

### 1. TPE (Tree-structured Parzen Estimator)
- **类型**：贝叶斯优化算法
- **特点**：
  - 基于概率模型的智能搜索
  - 平衡探索和利用
  - 适用于大多数场景
- **适用场景**：
  - 一般超参数优化任务
  - 搜索空间较大的情况
  - 计算资源有限的环境
- **默认参数**：
  - `n_startup_trials=5`：前5次试验使用随机搜索
  - `n_ei_candidates=24`：每次迭代考虑24个候选点

### 2. CMA-ES (Covariance Matrix Adaptation Evolution Strategy)
- **类型**：进化算法
- **特点**：
  - 基于种群的全局优化
  - 适合连续参数优化
  - 对噪声不敏感
- **适用场景**：
  - 连续参数优化
  - 高维搜索空间
  - 需要全局最优解的场景

### 3. GP (Gaussian Process)
- **类型**：贝叶斯优化算法
- **特点**：
  - 基于高斯过程的概率模型
  - 提供不确定性估计
  - 精度高但计算开销大
- **适用场景**：
  - 小规模搜索空间
  - 需要高精度的场景
  - 计算资源充足的环境
- **注意**：需要安装 scikit-learn 库

### 4. Random (随机搜索)
- **类型**：随机搜索
- **特点**：
  - 简单直接
  - 并行化效率高
  - 对某些场景表现意外地好
- **适用场景**：
  - 快速初步搜索
  - 高度并行化环境
  - 作为其他算法的补充

## 算法选择最佳实践

### 推荐算法选择

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| 一般场景 | TPE | 平衡效率和效果，适用于大多数情况 |
| 连续参数 | CMA-ES | 对连续参数优化效果更好 |
| 高精度需求 | GP | 提供最准确的结果，但计算开销大 |
| 快速搜索 | Random | 并行化效率高，适合初步搜索 |
| 计算资源充足 | TPE 或 GP | 可以充分利用计算资源获得更好的结果 |
| 计算资源有限 | TPE 或 Random | 效率更高，更快获得可行解 |

### 并行化设置

- **n_jobs=-1**：使用所有可用的CPU核心（推荐）
- **n_jobs=1**：串行执行，适合调试
- **n_jobs=N**：使用N个CPU核心，适合控制资源使用

### 剪枝策略

系统使用了 MedianPruner 剪枝策略，自动终止表现不佳的试验：
- **n_startup_trials=5**：前5次试验不剪枝
- **n_warmup_steps=3**：每个试验前3步不剪枝
- **interval_steps=2**：每2步检查一次是否需要剪枝

剪枝策略可以显著减少计算时间，特别是对于表现不佳的参数组合。

## 搜索空间优化

### 优化后的搜索空间

| 参数 | 取值范围 | 说明 |
|------|---------|------|
| learning_rate | [1e-5, 3e-5] | 使用对数空间搜索，更适合学习率调优 |
| batch_size | [16, 32] | 减少批大小选项，提高搜索效率 |
| epochs | [3] | 固定训练轮数，减少不必要的搜索 |
| dropout_rate | [0.1, 0.2] | 减少dropout选项，聚焦于常用值 |

### 搜索空间调整建议

1. **学习率**：
   - 对于BERT类模型，推荐范围：1e-5 到 5e-5
   - 对于小型模型，推荐范围：1e-4 到 1e-3

2. **批次大小**：
   - 取决于GPU内存，推荐：16-32
   - 内存充足时使用较大批次大小

3. **训练轮数**：
   - 对于微调任务，3-5轮通常足够
   - 对于从头训练，可能需要更多轮数

4. **Dropout率**：
   - 推荐范围：0.1-0.3
   - 防止过拟合，但过大可能导致欠拟合

## 使用示例

### 基本使用

1. 在训练配置对话框中：
   - 勾选"启用超参数自动寻优"
   - 选择"贝叶斯优化"作为寻优方法
   - 选择合适的优化算法（推荐TPE）
   - 设置并行作业数（推荐-1）

2. 点击"开始训练"按钮，系统会自动：
   - 执行超参数优化
   - 找到最优参数组合
   - 使用最优参数进行最终训练

### 高级配置

#### 命令行使用示例

```python
from hyperparameter_optimizer import HyperparameterOptimizer
from bert_dataset import create_dataset_from_standard_answers

# 加载训练数据
training_data = load_training_data()
dataset = create_dataset_from_standard_answers(training_data)

# 初始化优化器
optimizer = HyperparameterOptimizer(model_manager)

# 运行贝叶斯优化
result = optimizer.bayesian_optimization(
    dataset=dataset,
    n_trials=20,           # 搜索次数
    cv_folds=2,            # 交叉验证折数
    algorithm='tpe',        # 优化算法
    n_jobs=-1              # 使用所有CPU核心
)

# 获取最优参数
best_params = result['best_params']
best_score = result['best_score']

print(f"最优参数: {best_params}")
print(f"最优分数: {best_score:.4f}")
```

## 性能优化建议

1. **使用并行化**：
   - 始终设置 `n_jobs=-1` 以使用所有CPU核心
   - 对于大型搜索空间，并行化可以显著减少时间

2. **合理设置搜索空间**：
   - 减少参数组合数量
   - 选择合理的参数范围
   - 对于连续参数，使用范围搜索而非离散值

3. **选择合适的算法**：
   - 一般场景使用TPE
   - 连续参数使用CMA-ES
   - 快速搜索使用Random

4. **利用剪枝策略**：
   - 剪枝可以自动终止表现不佳的试验
   - 减少计算时间，提高搜索效率

## 故障排除

### 常见问题及解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Optuna库未安装 | 缺少依赖 | 运行 `pip install optuna` |
| GP算法失败 | 缺少scikit-learn | 运行 `pip install scikit-learn` |
| 并行化不工作 | 资源限制 | 检查系统资源，减少并行度 |
| 搜索速度慢 | 搜索空间过大 | 优化搜索空间，减少参数组合 |
| 寻优结果不佳 | 搜索次数不足 | 增加n_trials参数 |

### 性能问题排查

1. **检查CPU核心数**：
   - 确保系统有足够的CPU核心
   - 并行化效果与核心数成正比

2. **检查内存使用**：
   - 并行化会增加内存使用
   - 内存不足时减少并行度

3. **调整搜索参数**：
   - 减少n_trials可以加快搜索速度
   - 减少cv_folds可以减少计算量

## 结论

超参数优化是提高模型性能的重要手段，通过选择合适的优化算法和配置，可以显著提高模型训练的效率和质量。本系统提供了多种先进的优化算法，结合剪枝策略和并行化搜索，为用户提供了高效、灵活的超参数优化解决方案。

建议用户根据具体场景选择合适的优化算法，并通过调整搜索空间和并行度来平衡搜索速度和结果质量。
